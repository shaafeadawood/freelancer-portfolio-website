# freelancer-portfolio-website

Professional freelancer portfolio website (boilerplate) — Git Flow ready.

Repository structure (created):

- public/
  - index.html
  - portfolio.html
  - services.html
  - blog.html
  - contact.html
  - assets/
    - css/
      - style.css
    - img/
- .gitignore

Quick start

1. Work on `develop` branch (Git Flow):

```bash
git checkout develop
```

2. Create a feature branch, work, push, open PR against `develop`:

```bash
git checkout -b feature/your-feature-name
git add .
git commit -m "feat: add ..."
git push -u origin feature/your-feature-name
# Open a Pull Request on GitHub merging feature/* → develop
```

Notes

- This repository uses repository-level Git config for author identity.
- Default branch for development is `develop`. A local `develop` branch has been created and pushed; change the remote default on GitHub if needed.
